# Security Policy

## Supported Versions

Only the latest release and the current development on the `main` branch are actively maintained for security fixes.

## Reporting a Vulnerability

The project team and community take security seriously. If you believe you have found a security vulnerability in egakareta, please report it to us as soon as possible.

### Reporting via GitHub (Preferred)

We prefer that security vulnerabilities be reported using GitHub's **Private Vulnerability Reporting**. This allows us to collaborate on a fix in private before disclosing it publicly.

1. Navigate to the [egakareta repository](https://github.com/egakareta/egakareta) on GitHub.
2. Click on the **Security** tab.
3. Click on **Advisories** in the left sidebar.
4. Click **Report a vulnerability**.

### Reporting via Email

If you cannot use GitHub's reporting tool, you can report vulnerabilities by sending an email to team@egakareta.com. Please include as much detail as possible, including reproduction steps.

## Disclosure Process

When a vulnerability is reported:

1. We will acknowledge receipt of the report within 48-72 hours.
2. We will investigate the issue and confirm the vulnerability.
3. If confirmed, we will work on a fix in a private branch/advisory.
4. We will coordinate a disclosure date with you, ideally once a fix is available and a new version is released.

Thank you for helping keep egakareta secure.
